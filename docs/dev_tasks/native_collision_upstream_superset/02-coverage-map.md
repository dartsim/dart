## Mapping Rules

- Every in-scope row from `01-upstream-inventory.md` must resolve to one of:
  `covered`, `new-test-needed`, `new-benchmark-needed`, `fixture-needed`, or
  `not-applicable`.
- `not-applicable` requires a concrete reason, such as a non-collision Bullet
  dynamics test or an ODE joint solver test outside native collision detector
  behavior.
- Existing DART coverage may satisfy an upstream case only when the checked
  geometry, transform, query type, expected result, and edge condition are
  materially equivalent.
- Correctness coverage gates performance work. Benchmarks added from this map
  must not be used for optimization claims until their corresponding
  correctness fixtures are covered.
